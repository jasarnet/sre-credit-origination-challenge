# Technical Challenge – Site Reliability Engineer

Este repositorio contiene la documentación de diseño para una estrategia de
confiabilidad y observabilidad aplicada a una API crítica de negocio.

El foco de este challenge es:
- Definir señales objetivas de confiabilidad
- Alinear el comportamiento del sistema con el impacto al negocio
- Establecer una base sólida para la operación en producción

Los detalles de implementación se dejan fuera de alcance de forma intencional.
