## Objetivo

Diseñar y demostrar una estrategia de **confiabilidad y observabilidad** para una API crítica de negocio, que permita:

- Medir de forma objetiva la experiencia real del usuario.
- Detectar degradaciones antes de que se conviertan en incidentes graves.
- Tomar decisiones operativas basadas en SLOs y *error budgets*.
- Reducir el impacto de fallas en dependencias internas y externas.
- Estandarizar la observabilidad mediante automatización (IaC).

El objetivo es garantizar que la API de originación de créditos sea **predecible, medible y operable bajo presión**, incluso cuando las cosas fallan.

> No que nunca falle, sino que falle de forma **controlada, visible y recuperable**.
